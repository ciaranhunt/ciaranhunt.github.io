前言

学完SSM框架后，不太想急于学习新的东西，感觉自己好多掌握的不太好，再学新的知识就是纸上谈兵了，就决定先写一个项目，挑选了半天，大部分都是电商项目，感觉都太普遍了，就挑了一个博客系统，正好自己也弄了几天的博客，很有兴趣，就决定是他了。

1. 介绍

项目分为前台和后台(不是前端和后端)，感觉前台依赖与后台，所以先写后台。原来的项目是这样的：

决定先不写评论，其它小细节也略有删改，大体上就是这样了，直接用的layui模板。

1.2 页面裁定

  • 仪表盘页面：排行榜与新用户取消，展示为最新文章
  • 文章页面：标签页面删除
  • 附件页面：保留
  • 用户管理：第三方先不做

1.3 数据库设计

1.3.1 用户表

  • id：主键，自增
  • name：用户名
  • passwd：密码
  • email：邮箱地址
  • avatar：头像地址
  • info:个人说明
  • roleid：角色的id

1.3.2 角色表

  • id：主键，自增
  • name：角色名
  • rightid：权限id

1.3.3 权限表

  • id：主键，自增
  • name：权限名

1.3.4 文章表

  • id：主键，自增
  • uid:用户id
  • title:标题
  • path：文件路径
  • createTime：创建日期
  • modifyTIme：最近修改日期
  • cid:分类id（set集合）

1.3.5文章分类表

  • id：主键，自增
  • name：分类名
