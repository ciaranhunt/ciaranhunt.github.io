前言

Spring MVC属于SpringFrameWork的后续产品，已经融合在Spring Web Flow里面。Spring 框架提供了构建 Web 应用程序的全功能 MVC 模块。

1.介绍

1.1 核心组件

  • DispatcherServlet：前置控制器，接收所有请求(如果配置‘/’的话不包含jsp)
  • HandlerMapping接口：处理请求的映射
  • HandlerAdapter接口：调用具体的方法
  • ViewResolver接口：视图解析器，解析结果，准备跳转到具体视图

1.5 参数

1.5.1 普通参数

以后基本用注解了，下面也是基于注解方式的

如果要接收一些表单数据，可以直接在方法参数列表添加即可，不用管数据类型。比如数据表单中input type="text" name="age"，可以直接在方法中添加参数int age或者直接是包含此属性的对象People。

如果名称不一致时，可使用注解来解决。比如前端传参为name1，而对象的参数名为name，则使用@RequestParam(value = "name1'") String name。

  • @RequestParam(defaultValue = "value")可以给默认值，解决没有值穿过来时，不报错
  • @RequestParam(required = true)必须有值，不可为空

1.5.2 对于复杂参数

比如多选框，可以使用‘ @RequestParam(“name”) List list’来接收。
