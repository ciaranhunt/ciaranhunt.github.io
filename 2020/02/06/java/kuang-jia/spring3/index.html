5. 自动注入

在spring配置文件中，对象名与引用名相同，可以使用自动注入，不需要配置property

5.1 使用

在bean标签添加autowire属性，可取值为：

  • default：使用全局的配置(在beans标签内的属性default-autowire)
  • byName：通过属性名与bean的id对应
  • byType：通过属性类型与bean对应
  • no：不自动注入
  • constructor：通过构造器，构造器的参数名必须与bean的id一致

注意：sqlSessionFactory最好不要使用自动注入

7.2 read-only

  • true：告诉数据库为只读事务(可能提高效率)
  • false：(默认值)告诉数据库为提交事务

7.3 propagation

控制事务传播行为；当一个具有事务控制的方法A被另一个具有事务控制的方法B调用时，需要如何管理事务A？

  • REQUIRED：(默认值)，如果当前有事务，就在当前事务中执行；如果没有，则新建事务。
  • SUPPORTS：如果当前有事务，就在当前事务中执行；如果没有，就在非事务状态下执行。
  • MANDATORY：必须在事务内部执行，如果当前有事务，就在事务中执行；如果没有，报错。
  • REQUIRES_NEW：如果当前没有事务，就新建事务；如果当前有事务，就将当前事务挂起，新建事务去执行。
  • NOT_SUPPORTED：必须在非事务状态下执行，有则挂起。
  • NEVER：必须在非事务状态下执行，有则报错。
  • NESTED：必须在非事务状态下执行，有则创建一个嵌套事务。

7.4 isolation

在多线程或并发访问下如何保证访问的数据的完整性？

脏读，不可重复读，幻读

  • DEFAULT：(默认值)有底层数据库自动判断应该使用什么数据库
  • READ_UNCOMMITTED：读取未提交数据，可能出现脏读，不可重复读，幻读，效率最高
  • READ_COMMITTED：只能读取其它事务提交的数据，可以防止脏读，可能出现不可重复读，幻读
  • REPEATABLE_READ：读取的数据加锁，防止其它事务修改此数据，可以防止脏读，不可重复读，可能出现幻读
  • SERIALIZABLE：排队等待，对整个表加锁，最安全的，效率最低，可以防止脏读，不可重复读，幻读

8. 常用注解

  • @Component：创建类对象，相当于bean标签
  • @Service：与上面功能一致，写在serviceimpl类上
  • @Repository：与上面功能一致，写在数据访问层类上
  • @Controller：与上面功能一致，写在控制层类上
  • @Autowired：spring提供的自动注入(不需要写get/set方法)
  • @Value:获取属性文件的内容
  • @Pointcut：定义切点
  • @@Aspect：定义切面
  • @Before()：前置通知
  • @After()：后置通知
  • @AfterReturning()：后置通知
  • @AfterThrowing：异常通知
  • @Around()：环绕通知
