4.AOP(面向切面编程)

4.1 基本概念

正常程序都是纵向执行流程，即按顺序执行。

面向切面编程，在原有纵向执行流程中添加横切面。不需要修改原有程序代码，体现了程序高扩展性。

利用AOP可以对业务逻辑的各个部分进行隔离，从而使得业务逻辑各部分之间的耦合度降低，提高程序的可重用性，同时提高了开发的效率。

  • 切点：原有功能
  • 前置通知：在切点之前执行的功能
  • 后置通知：在切点之后执行的功能
  • 切面：所有功能的总称

如果切点执行过程中出现异常，会触发异常通知

4.2 搭建步骤

  1. 除了核心包之外，还需要spring-aop.jar、spring-aspects.jar、spring-tx.jar，同时spring-aop.jar还依赖于aspectjweaver.jar

  2. spring配置文件添加xsd

    xmlns:aop=”http://www.springframework.org/schema/aop"
    http://www.springframework.org/schema/aop
    https://www.springframework.org/schema/aop/spring-aop.xsd

spring提供了两种AOP实现方式Schema-based和AspectJ。

前置通知实现MethodBeforeAdvice；后置通知实现AfterReturningAdvice。

  • 实现前置通知方法的参数：
    • method：切点方法对象
    • object[]：切点方法参数
    • object:切点方法所在类对象

4.2.2 AspectJ方式

通知类不需要实现接口，配置都在<aop:config>的子标签<aop:aspect>中,可以将所有通知都写到一个类中，只需要在配置文件配置即可.

注意：

  • <aop:after>:切点有异常也执行。

  • <aop:after-returning>:有异常不执行，切点正常才执行。

  • 获取切点参数：同时在pointcur标签中配置expression属性和advisor标签配置arg-names属性,不可使用通配符。

4.3 注解使用(基于Aspect)

spring不会自动去寻找注解，需要在配置文件中注明。

  1. 配置xsd

    xml

  2. 配置要扫描的包

    xml
    • ture：使用cglib动态代理
    • false：使用jdk动态代理

  3. 注解

  • 替代bean标签，在类上写@Component。如果无参，直接类名首字母小写；@Component(“自定义名称”)

  • 替代aop:pointcut标签：在方法上面写@Pointcut(“top.ciaranhunt.test.Demo.demo1()”)

  • 通知类上写@@Aspect，通知类的方法写：

    • 前置通知：@Before(“top.ciaranhunt.test.Demo.demo1()”)
    • 后置通知：@After(“top.ciaranhunt.test.Demo.demo1()”)或者 @AfterReturning (“top.ciaranhunt.test.Demo.demo1()”)
    • 异常通知：@AfterThrowing(“top.ciaranhunt.test.Demo.demo1()”)
    • 环绕通知：@Around(“top.ciaranhunt.test.Demo.demo1()”)
