5. 动态sql

根据不同的条件需要执行不同的sql命令，称为动态sql。即在mapper.xml中添加判断。

5.1 if

5.2 where

where元素只会在至少有一个子元素的条件返回SQL子句的情况下才去插入“WHERE”子句。而且，若语句的开头为“AND”或“OR”，where 元素也会将它们去除。

xml

5.3choose, when, otherwise

有时我们不想应用到所有的条件语句，而只想从中择其一项。针对这种情况，MyBatis 提供了choose元素，它有点像Java中的switch语句。

xml

5.4set

set元素会动态前置SET关键字，同时也会删掉无关的逗号，因为用了条件语句之后很可能就会在生成的SQL语句的后面留下这些逗号。

7. 缓存

应用程序与数据库交互的过程是一个相对耗时的过程，减少对数据库的访问，加快应用程序的速度。

mybatis中默认sqlsession开启缓存，同一个sqlsession调用同一个select，第一次查询后将结果缓存到内存中。

缓存的是statement对象，一个select标签对应一个statement对象。

使用二级缓存，即SqlSessionFactory缓存

在mapper.xml中配置cache标签

xml

sqlsession不关闭或这不提交，都不将缓存移到二级缓存。

8. 实现多表查询

多表查询时，一个类包含另一个类的对象：

  • 单个对象
  • 多个对象(List集合)

8.1 业务装配

对两个表进行单表查询，将查询结果关联起来。(一个对象包含另一个对象)

8.2 Auto Mapping特性

使用别名结合Auto Mapping特性实现多表查询。只适用于单个对象，对于集合对象不行。

比如Student中包含Teacher对象，可以使用teacher.id别名(此处需要转义，使用反单引号)

8.3 resultMap标签

编写在mapper.xml。有程序员控制ssql查询结果与实体类对象的映射关系。默认mybatis使用Auto Mapping特性，查询结果与实体类字段名称相同。

使用resultMap标签时，select标签不使用resultType属性，而是使用resultMap属性引用resultMap标签。

主键使用id标签配置。其它列使用result标签配置。

type表示返回值类型；column表示列名，property表示属性名。

xml

如果resultMap中需要关联一个对象时，可使用association标签。

使用N+1方式：
property表示属性名；select表示调用其它xml的select的id；column表示需要传递的当前表列值。

xml

如果resultMap中需要关联集合对象时，可使用collection标签。多一个ofType属性，表示泛型的类型。

如果为联合查询，可以把association标签和collection标签当成一个小型的resultMap标签

9. 运行原理

9.1 运行过程涉及的类

  1. Reasources Mybatis中IO流的工具类 —加载配置文件

  4. Configuration —封装了全局配置文件的所有配置信息
  5. Transaction事务类 —每一个SqlSession都有一个事务类对象
