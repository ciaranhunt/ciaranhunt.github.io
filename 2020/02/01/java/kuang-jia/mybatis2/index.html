4. 接口绑定方案及多参数传递

4.1 接口绑定

作用：实现创建一个接口后吧mapper.xml有mybatis生成接口的实现类，通过调用接口对象就可以获取mapper.xml编写的sql。

与spring整合时使用这个方案

实现步骤：

  1. 创建一个接口
    • 要求包名和接口名与mapper.xml中namespace一致。
    • 要求方法名与id属性一致。
  2. 在全局配置文件中使用package进行扫描接口和mapp.xml
  3. 在类中使用，具体如下：（LogMapper为接口，selAll为接口中的方法）

接口：为什么可以实例化？

  • 使用的jdk的动态代理模式。

5.2 where

where元素只会在至少有一个子元素的条件返回SQL子句的情况下才去插入“WHERE”子句。而且，若语句的开头为“AND”或“OR”，where 元素也会将它们去除。

5.3choose, when, otherwise

有时我们不想应用到所有的条件语句，而只想从中择其一项。针对这种情况，MyBatis 提供了choose元素，它有点像Java中的switch语句。

xml

5.4set

set元素会动态前置SET关键字，同时也会删掉无关的逗号，因为用了条件语句之后很可能就会在生成的SQL语句的后面留下这些逗号。
