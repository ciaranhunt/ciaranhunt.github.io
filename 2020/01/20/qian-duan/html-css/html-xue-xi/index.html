前言

HTML5 将成为 HTML、XHTML 以及 HTML DOM 的新标准。

HTML 的上一个版本诞生于 1999 年。自从那以后，Web 世界已经经历了巨变。

HTML5 仍处于完善之中。然而，大部分现代浏览器已经具备了某些 HTML5 支持。

1. 基础

1.1 实体

在 HTML 中，某些字符是预留的。

在 HTML 中不能使用小于号（<）和大于号（>），这是因为浏览器会误认为它们是标签。

如果希望正确地显示预留字符，我们必须在 HTML 源代码中使用字符实体（character entities）。
也就是转义。

语法：&实体的名字；
常用的实体：  
    空格---&nbsp;
    小于号---&lt;
    大于号---&gt;
    版权（copyright）---&copy;

1.2 图片-img标签

<img src="https://s2.ax1x.com/2020/01/04/ldzewq.md.jpg" alt="ldzewq.jpg" border="0" />

属性：
- src：指定图片地址
- alt：指定图片描述
- width：修改图片宽度
- height：修改图片高度

1.3meta标签

作用：
1. 设定字符集
2. 描述网页内容，设定关键字
3. 请求重定向

1.4 XHTML语法规范

文档结构
    XHTML DOCTYPE 是强制性的
    <html> 中的 XML namespace 属性是强制性的
    <html>、<head>、<title> 以及 <body> 也是强制性的
元素语法
    XHTML 元素必须正确嵌套
    XHTML 元素必须始终关闭
    XHTML 元素必须小写
    XHTML 文档必须有一个根元素
属性语法
    XHTML 属性必须使用小写
    XHTML 属性值必须用引号包围
    XHTML 属性最小化也是禁止的

2. 核心技术

2.1 attr与prop

  • attribute
    包括html预定义属性与html自定义属性,是HTML标签的属性。

  • property

    是js原生对象的直接属性。

每一个预定义的attribute都有一个property对象与之对应。

  • 对于非布尔值属性，不管什么情况 property和attribute都会同步。

  • 对于布尔值属性

    1. 改变 property时,不会同步attribute
    2. 在没有动过property时attribute会同步property，一旦动过property，attribute不会同步property

浏览器与用户操作的都是property

那么，什么时候使用attr()，什么时候使用prop()

1、是有true,false两个属性值使用prop()，prop多适用于复选框全选和反选的、下拉框、禁用、单选情况。

2、其他属性操作则使用attr();

2.2 H5小功能

  • 若标签的class属性有多个值时，可以使用classList获取class数组，使用add（）或remove（）方法添加或删除class值；toggle()方法无则新增，有则删除。
  <body>
      <div id="t0" class="t1 t2 t3" data-hl="hl">
      </div>
  </body>
  var tNode=document.getElementById("t0");
  console.log(tNode.classList);
  tNode.classList.add("t4");
  tNode.classList.remove("t1");
  console.log(tNode.classList);
  tNode.classList.toggle("t1");
  console.log(tNode.classList);

控制台输出结果为：

  • 对于标签中自定义属性，使用data-*的格式可以使用dataset获取。

     console.log(tNode.dataset.hl);

    输出结果为：

    hl

  • 可编程能力。可以对文本进行编辑，在标签属性添加contenteditable=”true”

    <div id="t0" contenteditable="true">
    hello world!
</div>

  • H5开头的表示处于标准模式；如果不在开头或者没有，则处于怪异模式
